Axis overlap

Adjusts the overlap (or gaps) between the four cardinal directions (Up, Down, Left, Right).

At 0%: The 360º circle is divided perfectly into four 90º pizza slices, leaving no overlaps and no gaps.

Above 0%: The slices become larger than 90º, causing them to overlap. When the thumbstick is within an overlap zone, both adjacent directions are triggered simultaneously.

Below 0%: The slices become smaller than 90º, creating empty gaps between them. When the thumbstick is within a gap, no direction is triggered.

Deadzone

The central circular area in which thumbstick movement registers no input.

It is recommended to set this deadzone as low as possible (depending on how tight the centering spring is) and then set your in-game deadzone to the minimum allowed.

Setting this value to 0% is guaranteed to cause drift, as this is a hardware-level adjustment.

Override global deadzone: Use a custom deadzone value for this profile, instead of the value defined in global settings.

Anti-deadzone

Introduces a sharp output ramp at the edge of the deadzone, causing the initial output value to start above 0%.

The purpose of this setting is to counteract and neutralize forced in-game deadzones that cannot be disabled otherwise.

Outer threshold

The circular threshold that divides what is considered inner ring vs outer ring.

Thumbstick input below this threshold (and above the deadzone) triggers the inner ring action defined in the profile.

Thumbstick input above this threshold triggers the outer ring action defined in the profile.

Upper limit

Saturation setting that defines the sensitivity of the output.

100%: Adjusted sensitivity to match the reference design thumbstick modules and top case.

Below 100%: Requires smaller movements to reach max outout, so it makes an outer deadzone.

Above 100%: Requires larger movements to reach max output.
(Could be useful for custom thumbstick modules or modified top cases)

Acceleration curve

Achieves non-linear output by applying a Felix curve (a Schlick bias function tuned for input purposes) to the output.

Exactly 0: Linear response.

Below 0: A more relaxed curve that requires larger movements to achieve the same results.

Above 0: A more aggressive curve that requires smaller movements to achieve the same results.

Vertical sensitivity ratio

Increase or decrease the sensitivity of the actions mapped to the Y axis, in relation to the overall sensitivity defined for the thumbstick.

100%: Same response for X and Y axis.

Below and above 100%: Less or more responsive Y axis.

Sensitivity as mouse

The maximum output in pixels per second (for when the thumbstick is at the edge).

This setting applies only if mouse axis are mapped to the thumbstick.

Sensitivity as scrollwheel

The maximum output in ticks per second (when the thumbstick is at the edge).

This setting applies only if scrollwheel axis are mapped to the thumbstick.

Extras

Push auto-toggle: Keeps the push action engaged while the thumbstick is tilted in any direction, and automatically toggles it off when the thumbstick goes back to the center.

(Ideal to emulate console-like sprint toggle behavior).

Axis self-align: Calculates the output for each axis based on the absolute distance from the center (combining both X and Y components) rather than calculating each axis independently. This prevents the player from having to align their movements perfectly with the cardinal directions to achieve the expected input.

(Ideal for using gas and brake on the thumbstick while tilting the controller for steering).

Center deadzone

The central circular area in which thumbstick movement registers no input.

For rotation mode, a larger deadzone is often ideal. This ensures the stick only reacts when sweeping the outer perimeter, preventing unexpected inputs as it snaps back to the center.

Entry deadzone

The angle around the entry point direction in which the input is ignored (or less relevant).

For gamepad axis, the output on the entry deadzone is 0%.

For mouse axis (flickstick), the deadzone ignores flicks.

Entry point

The direction that is assumed as the starting point (or zero) when sweeping around the perimeter.

Mapped action angle: Either up/down/left/right depending on the action mapped in the profile.

Any angle: Whatever angle the user deflects the stick from the center will be considered the starting point, without snapping into any cardinal direction.

Direction

The sweep motion direction in which the values increase, either clockwise [↻] or anti-clockwise [↺]

Negative variants of the axis actions behave as if this setting is inverted.

Sensitivity as mouse

The output in pixels for each complete (360º) thumbstick rotation.

Based on gyro RWS: When enabled, a complete thumbstick rotation becomes 1 RWS in comparison to the RWS reference for the gyro-to-mouse emulation. It makes flickstick configuration easier.

Sensitivity as scrollwheel

The output in scrollwheel steps for each complete (360º) thumbstick rotation.

Sensitivity as axis

The output in axis percentage for each complete (360º) thumbstick rotation.

This setting applies only for gamepad thumbsticks X/Y output, and gamepad triggers output.

Flick time

How long it takes for the mouse output to catch up with a flick (a sudden input away from the defined entry point).

The emulated sweep is logarithmic, it starts fast, but slows down gradually.

This setting applies only for mouse output, and is disabled if Entry point is set as "Any angle", because flicks cannot occur.

Value while in center

Output value while the stick is in the central deadzone area, and not being swept.

Zero: Output always defaults to 0% when the stick goes back to the center.

Keep last value: The last value registered during the sweep will continue to be sent, even when the stick returns to the center.

This setting is only enabled for gamepad axis output.

Smoothing

The maximum number of thumbstick samples aggregated to eliminate analog noise.

The smoothing is dynamic, so the number of samples is reduced the faster the movements is, to lower perceived latency.

This setting is independent from global thumbstick smoothing, as sweeping requires increased accuracy.
